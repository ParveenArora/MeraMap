Meramap-Api - What it Does

Introduction

Meramap-api is a debian package to install an OpenStreetMap API Server, also known as the Rails Port. Note that this means it is installing a copy of the OpenStreetMap Web Site, and the database interface to allow editing of OpenStreetMap Data stored locally on your own server. It does NOT generate map tiles to display on web pages - for that you need meramap.deb, not meramap-api.deb.

Unless you know what the above paragraph is talking about, you probably do not want to install meramap-api!!!

Meramap-api Components

Meramap-Api installs and configures the following components for you - it is basically following the Rails Port installation in structions.

  1. Dependency Checks - the Debian Package ensures that you have all the required packages installed on your system before it starts, and fails with an error if not.
  2. Configures postgresql database, creates databases and a user.
  3. Ruby - the ruby programming language, which conrols the OSM web site.
  4. Ruby Gems - the ruby library system that is used to add other components to ruby.
  5. The OSM Rails Port - this is downloaded from the OSM git repository, and configured automatically, as specified in /usr/local/meramap-api/scripts/variables.sh
  6. Osmosis - the tool to import openstreetmap data into the database.
  7. gpx-import - the deaemon that processes gpx files uploaded to the web site.
  8. It installs a small sample of OSM data into the database for testing.
  9. Starts the ruby rails development server to try out the new system.
